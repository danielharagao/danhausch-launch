Sistema: Predictive History Graph Explorer

Objetivo: transformar dados históricos e estruturais em leitura de trajetórias prováveis (não adivinhação).

Abrir interface de grafos

1) Como pensar o modelo

  • Nós = pessoas, supernós (agregações), blocos.
  • Arestas = relações (cooperação, rivalidade, dependência, influência).
  • Camadas = people / supernodes / blocks.
  • Regra de ouro: o modelo estima probabilidades condicionais, não eventos exatos.

2) Fluxo de uso (rápido)

  1. Acesse /ph-graph/.
  2. Escolha a camada: people, supernodes ou blocks.
  3. Use zoom/pan para explorar clusters.
  4. Clique em um nó para ver conexões e atributos no painel lateral.
  5. Compare densidade e centralidade visual entre camadas.

3) Leitura para historiadores

  • Busque continuidade estrutural: quais blocos permanecem centrais ao longo do tempo?
  • Observe rupturas: perda súbita de conexões de um nó-ponte.
  • Compare padrão atual com ciclos anteriores (pré-crise, crise, recomposição).
  • Use supernós para síntese macro; volte para people para evidência micro.

4) Leitura para futuristas

  • Mapeie sinais fracos: nós periféricos ganhando conexões rapidamente.
  • Teste hipóteses: “se nó X perder ponte com Y, que bloco absorve influência?”
  • Monitore fragilidade: alta dependência de poucos nós críticos.
  • Combine visual de rede + drivers de risco no War Room (/ph-app/).

5) Protocolo analítico recomendado (5 passos)

  1. Descrever estado da rede (nós, arestas, densidade).
  2. Diagnosticar atores-ponte e vulnerabilidades.
  3. Comparar com período histórico análogo.
  4. Projetar 2–3 cenários plausíveis.
  5. Decidir gatilhos de monitoramento e ação.

6) Limites e cautelas

  • Não tratar saída como profecia.
  • Relação visual forte ≠ causalidade automática.
  • Dados incompletos podem subestimar atores informais.
  • Atualize o dataset com frequência para evitar “inércia analítica”.

7) Checklist de sessão de análise

  • [ ] Definir pergunta (ex.: risco de ruptura de coalizão)
  • [ ] Escolher camada inicial (macro ou micro)
  • [ ] Identificar 3 nós críticos e 2 pontes
  • [ ] Registrar hipótese principal + hipótese alternativa
  • [ ] Definir indicadores de confirmação/negação
